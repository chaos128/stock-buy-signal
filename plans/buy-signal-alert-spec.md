# 매수 신호 알림 시스템 — 구현 스펙

## 목표
QQQ(및 향후 다른 종목)에 대해 여러 기술적 조건이 동시에 정렬될 때
텔레그램/이메일로 알림을 받는다. 단일 조건이 아니라 **여러 신호가 겹칠수록
신뢰도가 올라가는 점수제(scoring)** 로 판정한다.

> ⚠️ **범위 주의:** 진입 신호는 전략의 절반일 뿐이다. 이 알림은 "관심 신호"를
> 제공하는 것이지 매매 시스템이 아니다. 실제 운용에는 아래의 청산·리스크 관리와
> 백테스트 검증이 반드시 함께 있어야 한다.

---

## 신호 계층 (Layered Signals)

진입은 아래 4개 층이 겹칠 때만 의미가 있다. 각 층을 개별 boolean으로 계산하고
합산 점수로 알림 여부를 결정한다.

### 1. 장기추세 필터 (필수 게이트)
- 조건: `close > SMA(200)`
- 이 조건이 false면 나머지 신호와 무관하게 알림을 보내지 않는다
  (평균회귀 전략은 상승추세에서만 유효).

### 2. 단기 눌림 (Pullback)
- `close <= BollingerBand_lower(length=20, mult=2)` (하단 터치/이탈)
- `RSI(14) <= 40`
- 확인 캔들 옵션: 다음 봉이 BB 하단 안으로 재진입 or 종가 반등 시 신뢰도 +

### 3. 시장 환경 (VIX 레짐) — 상대 기준 사용
- ❌ 절대 기준(`VIX > 30`)은 쓰지 않는다. 2026-06 급락 때 40% 급등하고도
  30을 못 넘겨 신호를 놓쳤던 사례가 있음.
- ✅ 상대 기준:
  - `VIX > SMA(VIX, 20)` 로 급등 감지, 그리고
  - 스파이크 후 꺾임 확인 (당일 고점 대비 종가 되밀림 or 전일 대비 하락 전환)
- 보조: Fear & Greed 지수가 "Fear" 이하로 진입

### 4. 시장 폭 (Breadth) — 지수 ETF 전용, 선택
- A/D Line 강세 다이버전스 (가격 신저점 & A/D 저점 상승)
- 52주 신저점 종목 수 급감
- 200일선 위 종목 비율 반등
- TRIN(Arms) 급등 후 정상화
- ⚠️ 데이터 확보가 이 프로젝트의 가장 큰 난관 (아래 데이터 소스 참고)

### 5. 매크로 게이트 (수동 확인 — 자동화 안 함)
- 평균회귀는 "이 하락은 일시적 공포"라고 가정한다. 하락 이유가 **구조적 레짐
  전환**(금리 인하→인상 리프라이싱, 신용 이벤트 등)이면 모든 기술 신호가
  정렬돼도 떨어지는 칼날이다.
- 자동화가 어려우므로: 알림이 뜨면 **뉴스/매크로를 사람이 한 번 확인**하는
  체크 단계로 둔다. "왜 빠졌는가"가 일시적 노이즈면 진입, 구조적이면 보류.
- 알림 메시지에 "매크로 확인 필요" 문구를 항상 포함시킨다.

## 알림 규칙 (Scoring)
- 층 1(추세)은 게이트: false면 즉시 중단.
- 층 2/3/4에서 충족된 신호 개수를 점수화.
- 예: `score >= 2` 이면 "관심", `score >= 3` 이면 "강한 정렬 → 알림 발송".
- 알림 메시지에 어떤 신호가 켜졌는지, 각 지표 현재값을 함께 포함.

---

## 청산 · 리스크 관리 (진입만큼 중요 — 알림에 함께 표기)

이 알림은 매수 신호를 주지만, 그 매수가 유효하려면 아래가 세트로 정의돼야 한다.
알림 메시지에 진입가 기준의 손절/익절 후보 레벨을 계산해 함께 보낸다.

- **손절 (Stop):** 진입 봉 또는 최근 스윙 저점 아래 (예: BB 하단 이탈 시
  최근 N일 저점 - 여유). 손절 없이는 밴드워킹에 계속 물린다.
- **익절 (Target):** 1차 = 20일선(BB 중심선), 2차 = BB 상단. 평균회귀는
  "중심선 복귀"가 기본 목표.
- **손익비 (R:R):** 진입-손절 거리 대비 진입-1차익절 거리가 최소 1:1.5 이상일
  때만 유효 신호로 간주 (그 미만이면 알림에 "R:R 부족" 경고).
- **사이징 (Sizing):** score에 비례해 비중 차등 (예: score 2 → 절반,
  score 3+ → 전량). 계좌 대비 1회 리스크 상한(예: -1%)을 손절폭으로 역산.

## 실전 잡음 필터 (v1에 꼭 필요한 것들)

- **타임프레임:** 기본 **일봉(Daily)**. 견고함을 위해 주봉 추세 필터 +
  일봉 진입의 멀티 타임프레임 고려 (v2).
- **이벤트 캘린더 필터:** FOMC · CPI · 고용지표 발표 **직전 진입 회피**.
  QQQ는 금리 이벤트에 특히 민감. 캘린더 API로 D-1 신호는 보류/경고.
- **재알림 스로틀:** 급락장은 BB 하단을 며칠 연속 터치 → 매일 알림 폭탄.
  "동일 신호 N일(예: 5거래일) 내 재알림 금지" + 확인 캔들로 억제.
- **체결 갭 인식:** 알림은 **종가 기준**이라 그 가격에 못 살 수 있다.
  밤사이 갭 위험 → 알림은 "체결"이 아니라 "관심 신호"로만 취급 (메시지에 명시).

## 구현 경로 A — Python + 텔레그램 (권장: breadth·점수제까지 가능)

### 구조
1. **데이터 수집**
   - 가격/RSI/BB/VIX: `yfinance` (QQQ, ^VIX) — 무료로 충분
   - breadth: 별도 소스 필요 (아래 참고)
2. **조건 판정**
   - `pandas` + `pandas_ta` (또는 직접 계산) 으로 SMA200 / BB / RSI(14)
   - VIX 상대 기준 및 스파이크-꺾임 로직
   - 층별 boolean → score 합산
3. **알림 발송**
   - 텔레그램 Bot API (`sendMessage`) 또는 SMTP 이메일
   - 중복 알림 방지: 마지막 발송 상태를 파일/DB에 저장하고
     신호가 새로 켜진 순간에만 발송 (edge-trigger)
4. **스케줄 실행**
   - GitHub Actions cron (장중 N분 간격) — 무료, 서버 불필요
   - 또는 로컬 cron / 라즈베리파이

### breadth 데이터 소스 후보 (조사 필요)
- Nasdaq/NYSE breadth 심볼을 제공하는 데이터 벤더 API
- Stooq, Alpha Vantage 등 무료 API의 커버리지 확인
- 최악의 경우 breadth는 v2로 미루고 층 1~3만으로 v1 출시

### 파일 구조 예시
```
alert-system/
  fetch.py        # 데이터 수집
  signals.py      # 지표 계산 + 층별 판정
  notify.py       # 텔레그램/이메일 발송
  state.json      # 마지막 알림 상태 (중복 방지)
  main.py         # 오케스트레이션
  .github/workflows/alert.yml  # cron
```

## 구현 경로 B — TradingView Pine Script (빠른 시작)

- BB 하단 + RSI(14)<40 은 내장 지표로 즉시 alert 설정 가능
- VIX 등 타 심볼은 `request.security()` 로 끌어와 복합 조건 구성
- 하나의 alert 조건식으로 "추세 && 눌림 && VIX 상대급등" 표현
- 한계: 무료 요금제는 활성 알림 개수 제한(1~2개), breadth 다중 조합은 복잡

### Pine 복합 조건 뼈대
```pine
//@version=5
indicator("Buy Signal Alert", overlay=true)
sma200 = ta.sma(close, 200)
[bbMid, bbUp, bbLo] = ta.bb(close, 20, 2.0)
rsi = ta.rsi(close, 14)
vix = request.security("TVC:VIX", "D", close)
vixMa = request.security("TVC:VIX", "D", ta.sma(close, 20))

trendOk = close > sma200
pullback = close <= bbLo and rsi <= 40
vixSpike = vix > vixMa

buySignal = trendOk and pullback and vixSpike
alertcondition(buySignal, "BuySignal", "조건 정렬: BB하단+RSI+VIX")
plotshape(buySignal, style=shape.triangleup, location=location.belowbar)
```

## 수용 기준 (Acceptance Criteria)

### 로직 정확성
- [ ] 층 1~3 신호가 각각 정확히 계산된다
- [ ] 2026-06-08~10 구간에서 신호가 켜지는지, 2026-07 초 눌림에서는
      꺼져 있는지로 로직 sanity check (※ 이건 엣지 검증이 아니라 버그 확인용)

### 백테스트 검증 (엣지가 실재하는지 — curve-fitting 배제)
- [ ] 충분한 과거 구간에서 신호 진입 + 정의된 청산 규칙으로 시뮬레이션
- [ ] 지표 산출: **승률, 손익비(평균 R), 기대값, 최대낙폭(MDD), 거래 횟수**
- [ ] 파라미터(RSI 40, BB 2σ, SMA 200)는 임의값 → 인접 값에서도 성과가
      안정적인지 민감도 확인 (한 값에서만 좋으면 과최적화 의심)
- [ ] out-of-sample 구간에서 성과가 무너지지 않는지 확인

### 알림 동작
- [ ] 신호가 새로 켜진 순간에만 1회 알림 (재알림 스로틀 동작)
- [ ] 알림 메시지에 켜진 신호 목록 + 각 지표 현재값 + 손절/익절 후보 레벨
      + "매크로 확인 필요" 문구 포함
- [ ] 이벤트(FOMC/CPI/고용) D-1 신호는 보류 또는 경고 표기
